# Plan 02-01: App shell — `<RootLayout />`, full router, persistent `<Nav />`

## Goal

Extend Phase 1's two-route `createHashRouter` into the full D-24 table by introducing a single `<RootLayout />` route with 15 child leaf routes (one per v1.0 hash). Build a persistent `<Nav />` (profile picker, imports, compare, settings, about, theme toggle, language toggle), a `useScrollToTop()` hook for PUSH-navigation scroll restoration, and thin `<ThemeProvider />` / `<I18nProvider />` Context wrappers (D-14) that throw if consumed outside the tree. Every leaf route mounts a labelled placeholder component (`<WelcomePlaceholder />`, `<HomePlaceholder />`, etc.) so the shell is verifiable in isolation; plans 3–7 replace these placeholders with real views. This plan owns SHELL-01 (route table), SHELL-02 (hash deep links), SHELL-04 (provider tree), SHELL-05 (scroll-to-top). SHELL-03 (full `<Nav />` with shadcn `Sheet` mobile drawer) and SHELL-06 (toast + dialog primitives) ship in plan 2 — this plan delivers a desktop-only `<Nav />` shape with a stubbed hamburger button.
