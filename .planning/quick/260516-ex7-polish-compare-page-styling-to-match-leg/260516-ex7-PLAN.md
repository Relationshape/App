# Plan: Polish compare page styling + fix modal text size

## Context

The React compare page (`src/routes/Compare.tsx`) renders a stripped-down Tailwind UI that doesn't use the polished legacy CSS classes already shipped in `src/styles/legacy-components.css`. The page also lacks the "Kategorie-Details" cat-grid section the legacy app provides at `public/legacy/js/app.js#L3539-L3540`. Clicking a category card in the legacy app opens a tabbed modal showing the per-category spider chart — the React app has no equivalent component, and the existing `<ItemSpider>` renders axis labels at ~31px (way too large for a ~520px chart) without any line-wrapping for long labels.

This task ports the legacy compare layout into React (re-using the legacy CSS classes that already exist) and adds a `CategoryModal` component built on shadcn Dialog. The Edit-answers tab is explicitly out of scope.

## Task 1 — Rewrite `src/routes/Compare.tsx` to match legacy layout

**Files:** `src/routes/Compare.tsx`

**Action:**

1. Replace the existing JSX in the return statement with a structure that mirrors the legacy `viewCompare` shape:

## Task 3 — Fix `ItemSpider` label font size and add wrapping

**Files:** `src/components/charts/ItemSpider.tsx`, `src/lib/charts/math.ts`

**Action:**

1. **Add a `wrapLabel` export in math.ts** if it isn't already exported (note: a `wrapLabel` function exists at math.ts line 181, exported — good, no new code needed there).

2. **Modify `ItemSpider.tsx`:**
   - Reduce the default font size formula. The current `labelFontSize(items.length)` returns 18-34px (≈31px for 7 items). For ItemSpider in a modal context, this is too big. Two options:
     - **Option A (preferred):** Use a tighter formula inline: `const fs = Math.round(Math.max(11, Math.min(18, 130 / items.length)))` — gives ~18px for 7 items, ~14px for 9 items. This matches the screenshot's readable label size.
     - **Option B:** Add a `fontSize?: number` prop and pass `fontSize={14}` from CategoryModal.
   - **Go with Option A** — it's simpler and fixes the issue for all ItemSpider usages (Result drilldown also benefits).

3. **Add wrapping to label render block:**
   ```tsx
   const maxCharsPerLine = Math.max(8, Math.floor((fs * 0.55) === 0 ? 16 : (size / 4 / fs * 1.6)))  // rough heuristic: longer labels per line for smaller fonts
   // Simpler version: use 16 chars per line as a constant — labels wrap on word boundaries
   const lines = wrapLabel(displayItem, 16)
   ```
   Then render each label as:
   ```tsx
   <text key={`label-${i}`} x={lx} y={ly - ((lines.length - 1) * fs * 1.1) / 2} textAnchor={anchor} fontSize={fs}>
     {lines.map((line, li) => (
       <tspan key={li} x={lx} dy={li === 0 ? 0 : fs * 1.1}>{line}</tspan>
     ))}
   </text>
   ```
   Import `wrapLabel` from `@/lib/charts/math`.

4. **Adjust label padding:**
   - `const pad = Math.max(80, Math.ceil(fs * 4))` — current formula. After fontSize reduction, the pad will shrink correctly (40-72px). But with multi-line labels, we need a bit more vertical room. Bump to `Math.max(90, Math.ceil(fs * 5))` so wrapped labels don't overflow the SVG.

**Verify:**
- `bun run typecheck` passes
- `bun run test src/components/charts/__tests__/ItemSpider.test.tsx` — read the existing test; if it asserts a specific font size, update to the new value.
- Visual: open the category modal — confirm labels are smaller (~14-18px) and long labels like "Gemeinsamen Kommunikationsstil finden" wrap onto multiple lines instead of overflowing the chart.

**Done:** ItemSpider labels are 11-18px and wrap onto 2-3 lines for long text. Modal looks like the screenshot.
