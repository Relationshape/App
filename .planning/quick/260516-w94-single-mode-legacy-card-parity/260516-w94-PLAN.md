# Quick 260516-w94 — Single mode hero card parity

## Problem

After `260516-rm2`, both List and Single mode correctly scope to the active category, but Single mode renders the question with the **list-row** card (`RsQuestionCard` with `<strong>` title + `<textarea>` note). The legacy app renders a **hero card**:
- Big serif `<h1 class="q-card-item">` question title
- Compact cat header (`.q-card-cat`) **inside** the card (small icon + cat name + "Edit Item/Scale" link)
- ScalePicker inside `.q-card-slider`
- Single-line `<input class="q-card-note">`
- Back / Next inside `.q-card-actions`
- `.q-card-progress` counter pill (top-right, absolute)
- Colored top-bar via `--c: cat.color` (CSS already ships this — `border-top: 3px solid var(--c)` at `.q-card`)

CSS in `legacy-components.css:1002-1076` already covers everything. The component just needs to emit the legacy structure.

## Task 1 — Rewrite SingleMode render tree

**File:** `src/components/questionnaire/SingleMode.tsx`

Replace the current `<main>` + outer `.q-cat-head` + `RsScaleLegend` + `RsQuestionCard` layout with:

```tsx
<section className="page q-page q-single-page">
  <QuestionnaireHeader ... />          // keeps List/Single toggle + Results + ← Categories
  <p className="q-nav-hint muted small">{coarse ? mobileHint : desktopHint}</p>
  <div className="q-stack">
    {peekNext && <article className="q-card is-peek" style={`--c:${cat.color}`}>...</article>}
    <article className="q-card in" style={`--c:${cat.color}`} data-testid="single-card" {...bind()}>
      <div className="q-card-cat">
        <span className="q-card-icon" aria-hidden>{cat.icon}</span>
        <span>{catTitle}</span>
        {isCustom && <span className="q-item-tag">{t('custom_tag')}</span>}
        <Button variant="ghost" size="sm" onClick={onEditItemScale} data-testid={...}>
          {t('item_edit_scale')}
        </Button>
      </div>
      <h1 className="q-card-item">{item}</h1>
      <div className="q-card-slider">
        <ScalePicker .../>
      </div>
      <input
        className="q-card-note"
        type="text"
        placeholder={t('item_note_placeholder')}
        value={note}
        onChange={...}
        onBlur={commitNote}
        data-testid="single-note"
      />
      <div className="q-card-actions">
        <Button variant="ghost" disabled={cursor===0} onClick={prev} data-testid="single-back">
          ← {t('btn_previous')}
        </Button>
        <Button onClick={next} data-testid="single-next">
          {t('btn_next')} →
        </Button>
      </div>
      <div className="q-card-progress" data-testid="single-progress">
        {cursor + 1} / {items.length}
      </div>
    </article>
  </div>
  <QuestionnaireNav ... />
</section>
```

**Why drop `RsQuestionCard` for single mode:** RsQuestionCard's DOM is `<div class="q-item q-card">` with `<strong>` title — that's a list-row look. The legacy single card is a distinct hero layout. Keeping both modes routed through one component would force the component to branch internally and ship two unrelated trees. Cleaner: ListMode keeps RsQuestionCard, SingleMode inlines the hero card.

**Scale picker mutation logic:** Copy the `setScaleKey` + `clearAnswer` + `commitNote` handlers from RsQuestionCard (they mutate `result.answers[cat.id][item]` via `saveResult`). Plain functions; no extra components needed.

**Drop the chip scale legend (`RsScaleLegend`) from single mode** — legacy single never shows it (ScalePicker labels each tick).

**Acceptance:**
- `<article class="q-card">` is the sole question wrapper
- `q-card-cat`, `q-card-item`, `q-card-slider`, `q-card-note`, `q-card-actions`, `q-card-progress` all present
- Card style uses `--c` from `cat.color`
- Note is an `<input type="text">` (single line)
- ScalePicker present, marker reflects current `cell.scaleFrac`
- Back disabled at cursor=0; Next advances cursor and persists `progress.flatIndex`
- progress counter shows `${cursor+1} / ${items.length}` where `items.length` is items of THIS category only
