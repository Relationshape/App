# Quick 260516-w94: SingleMode hero-card legacy parity

## One-liner
Single mode now renders the legacy `.q-card` hero layout — `<h1 class="q-card-item">` title, in-card `.q-card-cat` header, `.q-card-slider` ScalePicker, single-line `<input class="q-card-note">`, `.q-card-actions` Back/Next, and the `.q-card-progress` counter pill — with the colored top-bar driven by `--c: cat.color`.

## What the user reported
> When I click a category, I go to the view where the "List/Single" toggle is. Clicking should go to that view, but should only show questions for the category I clicked on. Additionally, the UI must look just like the legacy UI (screenshot 1). Changing to "Single" should have a next/prev button, plus the option to add a note. The mood bar / selector we had must stay there. List = all non-hidden questions of the category in a list. Single = prev/next navigates all non-hidden questions of the category.

## What was already in place (260516-rm2)
- Both `ListMode` and `SingleMode` already resolve the active category from `enabledCats[clamp(progress.catIndex)]` and pull items via `enabledItemsForCat(result.answers, cat.id)` — neither uses `flatItemsForResult` anymore.
- `progress.flatIndex` is interpreted as "index within current category" — Single mode's prev/next stays within the clicked category.
- `RsQuestionCard` + `RsScaleLegend` exist for shared list-row use.

The remaining gap was purely visual: Single mode was reusing `RsQuestionCard` (a list-row layout with `<strong>` title + `<textarea>` note + outer `.q-cat-head`), which doesn't match the legacy hero card.

## What changed

### `src/components/questionnaire/SingleMode.tsx` — full render-tree rewrite
- Stopped rendering `RsQuestionCard` and `RsScaleLegend` inside Single mode.
- Inlined the legacy hero-card structure that exactly mirrors `viewQuestionnaireSingle` in `public/legacy/js/app.js:2517-2641`:
  - `<section className="page q-page q-single-page">` page wrapper.
  - `<p className="q-nav-hint muted small">` keyboard / swipe hint.
  - `<div className="q-stack">` positioning context (CSS already places cards via `position: absolute; inset: 0`).
  - `<article className="q-card is-peek" style={{ '--c': cat.color }}>` peek next card (suppressed under `useReducedMotion()`).
  - `<article className="q-card in single-card" style={{ '--c': cat.color }} data-testid="single-card">` — the active hero card.
- Inside the active card, in legacy order:
  - `.q-card-cat` — small cat icon (`.q-card-icon`) + cat title + optional `.q-item-tag` (custom) + ghost "Edit Item/Scale" button.
  - `<h1 className="q-card-item">{cur.item}</h1>` — the big serif question title.
  - `.q-card-slider` wrapping `<ScalePicker>` (the mood bar — continuous gradient with snapped key + `scaleFrac`).
  - `<input type="text" className="q-card-note">` — single-line note input (matches legacy; `defaultValue` + `onBlur` persists).
  - `.q-card-actions` — Back (`single-back`, disabled at cursor=0) + Next (`single-next`).
  - `.q-card-progress` — `{cursor + 1} / {items.length}` chip (absolute-positioned by CSS at top-right).
- Mutation handlers (`setScaleKey`, `clearAnswer`, `setNoteValue`) are inlined in the component — they mirror the ones that previously lived in `RsQuestionCard`, but typed for the SingleMode cursor.
- Preserved: swipe gestures (`useSwipe`), keyboard arrows (`useKeydown`), reduced-motion peek-suppression, edit-item-scale dialog placeholder.
- Removed: outer `.q-cat-head` block (the cat header now lives inside the card, per legacy parity) and the chip `RsScaleLegend` (legacy single doesn't show it — the scale picker labels each tick).
